### 5. Package Management

#### Problem
```txt
# requirements.txt exists but likely has loose versions
requests>=2.0
flask>=1.0
```

This means anyone installing today gets different versions than someone installing next month!

#### Solution
```txt
# Pin all dependencies exactly
requests==2.31.0
urllib3==2.1.0
certifi==2024.2.2
...
```

Or better yet, use `pip-tools`:
```
# requirements.in
requests>=2.31.0

# requirements.txt (auto-generated by pip-compile)
requests==2.31.0 \
    --hash=sha256:...
    --hash=sha256:...
```

### 6. Error Handling

#### Common Issues in Skills
1. **Silent failures** - If API call fails, script exits without explanation
2. **Generic error messages** - "Error" doesn't help users troubleshoot
3. **No retry logic** - Network blips cause complete failure

#### Standard Pattern Needed
```python
try:
    response = api_call()
except ApiError as e:
    logger.error(f"API call failed: {e}")
    if e.retryable and attempt < MAX_RETRIES:
        sleep(2 ** attempt)  # Exponential backoff
        continue
    raise
except Exception as e:
    logger.exception("Unexpected error")
    sys.exit(1)
```
